# Buy Sedifex Monorepo

Buy Sedifex contains a Firebase-backed marketplace with

- **Cloud Functions** to denormalize canonical store/product records into a public read model.
- **Next.js frontend** to render product, category, and store pages from `publicProducts`.

## Architecture at a glance

- Canonical store data lives in **`stores/{storeId}`**.
- Canonical product data lives in **`products/{productId}`** (flat collection).
- Public storefront data lives in **`publicProducts/{storeId}_{productId}`** and is generated by Cloud Functions.

### `publicProducts` schema (denormalized)

Typical document fields:

- Identity: `id`, `storeId`, `productId`
- Visibility: `isVisible`, `storeStatus`, `eligibleForBuy`, `buyOptOut`, `verified`
- Store fields: `storeName`, `storeSlug`, `storePhone`, `storeLogoUrl`, `storeBannerUrl`, `city`, `country`, `addressLine1`
- Product fields: `productName`, `productSlug`, `description`, `imageUrls`, `imageAlt`, `price`, `currency`, `featuredRank`, `itemType`, `shopLink`
- Commerce fields: `sku`, `barcode`, `taxRate`, `reorderPoint`, `stockCount`, `expiryDate`, `batchNumber`, `manufacturerName`, `productionDate`, `showOnReceipt`
- Contact fields: `waLink`
- Timestamps: `createdAt`, `productUpdatedAt`, `storeUpdatedAt`, `publishedAt`, `updatedAt`

## Local development

### Frontend

```bash
npm --prefix frontend install
npm --prefix frontend run dev
```

Copy `frontend/.env.example` to `frontend/.env.local` and set values.

### Cloud Functions

```bash
npm --prefix functions install
npm --prefix functions run build
npm --prefix functions run test
```

Copy `functions/.env.example` to `functions/.env` for local emulator usage.

### Firebase Emulator Suite (integration)

```bash
firebase emulators:start --only firestore,functions
```

## Backfill / migration scripts

To rebuild `publicProducts` from existing data:

```bash
ts-node scripts/backfill-public-products.ts --dry-run
# then
 ts-node scripts/backfill-public-products.ts
```

Use `--dry-run` first to inspect projected write/delete counts.

## Deployment

- Frontend: deploy from Vercel (root or `frontend/`).
- Firestore rules/indexes:

```bash
firebase deploy --only firestore:indexes,firestore:rules
```

- Cloud Functions:

```bash
firebase deploy --only functions
```
